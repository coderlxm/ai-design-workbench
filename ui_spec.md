# AI设计工作台 MVP（第一阶段）UI与交互规格说明书

本说明书专为 **Google Stitch / UI生成与交互设计工具** 提炼设计，聚焦于页面布局、交互组件、操作输入框及工作流界面状态，不包含任何后台技术细节或生图算法 Prompt 细节。

---

## 一、 页面整体布局架构

工作台采用顶部工具条结合**三栏式页面结构**布局：

### 1. 顶部工具条 (Top Bar)
展示及配置项目级通用参数：
* **项目名称展示**
* **产品主题输入**（单行文本框）
* **生成尺寸比例选择**（下拉菜单或单选组，包含：`1:1` / `4:5` / `9:16` / `16:9`）
* **模型选择**（下拉框，提供三个模型供选择：`豆包` / `gpt-image-2` / `gemini`）
* **“保存”按钮**

### 2. 左侧流程区 (Workflow Steps Area)
固定展示完整的流程进度树，每一步垂直排列，需显示以下状态与信息：
* **步骤编号与步骤名称**（第1步至第6步）
* **当前状态与是否完成**（如：未开始、处理中、已完成、失败）
* **操作控制标识**（是否可编辑、是否可重试）
* **当前步骤产物微缩缩略图**（生成后在此微型区域呈现缩略图）

### 3. 中间素材与主工作预览区 (Workspace & Preview Area)
核心主画布区域，内容随左侧当前选中的步骤动态切换展示：
* **场景输入视图**：模板图选择库、本地图片上传拖拽区、图片裁剪（Crop）操作视窗。
* **线稿提取视图**：提取出的极简线稿图大图预览区。
* **模特输入视图**：模特多视角参考图的卡片上传管理视窗，包含视角标签展示。
* **最终结果预览视图**：大图展示区域，包含“预览”、“下载”、“保存”、“重新生成”核心动作按钮。

### 4. 右侧 AI 工作流控制区 (AI Panel)
处理 AI 中间产物和控制流的专用面板：
* **任务执行状态面板**：显示当前正在执行 of AI 拆解任务状态与进度。
* **提示词编辑区域**：包含大文本框（Textarea），用于展示反推生成的“场景提示词”与“模特提示词”，并直接允许用户进行二次编辑修改。
* **错误与重试面板**：当步骤执行失败时，显示明确的错误原因文字，并提供醒目的“重试”动作按钮。

## 二、 六大交互步骤组件与输入规范

### 功能一：选择模板图片 / 上传人物场景图
* **交互组件**：
  * 图片拖拽上传区（支持上传 JPG、PNG、WEBP）。
  * 主体区域裁剪画布组件。
  * 场景设定（本期固定为“美国”，不可编辑状态）。
  * 提交生成按钮。

### 功能二：AI生成极简线稿图
* **交互组件**：
  * 大图预览画布（展示黑白线条的极简线稿图）。
  * “重新生成线稿”交互按钮。

### 功能三：AI逆向生成场景提示词
* **交互组件**：
  * 位于右侧面板的**可编辑多行文本框**。
  * 用于直接呈现和修改反推出来的中文场景提示词。

### 功能四：选择模特 / 上传模特六视图
* **交互组件**：
  * 多视角卡片上传列表：
    * 包含必填位：正面、左侧面、右侧面三个图片卡片插槽。
    * 包含建议/选填位：背面、半身、全身三个图片卡片插槽。
  * 图片卡片标签（展示自动识别或选定的视角标签）。
  * 卡片操作件：悬浮显示“删除”、“替换”、“重新上传”小按钮。
  * 肖像授权确认勾选框（Checkbox，用户必须勾选才可向下推进）。

### 功能五：AI逆向生成模特提示词
* **交互组件**：
  * 位于右侧面板的**可编辑多行文本框**。
  * “参考图特征不一致”警告提示条（Banner，仅在多视角特征冲突时浮现）。

### 功能六：组合最终 prompt 并调用模型生成图片
* **交互组件**：
  * 最终生成的广告图展示大画布。
  * 生成进度状态条。
  * 操作按钮组：“查看最终组合 Prompt”（弹窗或抽屉面板）、“下载效果图”、“保存效果图”、“重新生成（重试）”。

## 三、 视觉与风格规范

为了确保设计一致性，交互界面应当严格遵循以下视觉美学规范：
* **整体风格**：亮色风格 (Light Style)
* **系统主题色**：`#02b96b` (需作为主要按钮背景色、被激活与选中状态的进度节点高亮色、焦点输入框边框色、以及关键动作行为图标的主题高亮色)
